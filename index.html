Latest Stable . io

Makes it easy to be on "latest stable" with all your gem dependencies.

Workflow

LatestStable.io makes it dead simple to stay up to date with the upstream changes of your gem dependencies.

1. Connect with GitHub

After connecting your GitHub account we will watch your gem dependencies for upstream changes like security patches, new features and small bug fixes. However the best part is for public projects you can even skip the step of giving us access to your GitHub account as we only need the GitHub url of the project.

2. LatestStable will autoupdate your gem dependencies

Whenever our systems see updates in the gems your project depends on we go through the following steps.

  1. Fork your project
  2. Update the gems to the latest stable version
  3. Push those changes back to GitHub
  4. Create a pull request for the updated code

At this point your continuous integration system picks up the new pull request and runs your test suite.

3. Merge the pull request once tests are passing

All you have to do is hit the merge button for the pull request we created. Done! Your now on latest stable!

In case your tests are we split up the updates until we know which gems failing the test. Think of this process as an bundle update bisect that will tell you which dependency updates you can merge without problems and which of the gems needs some modification before tests pass cleanly.
